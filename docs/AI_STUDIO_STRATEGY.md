## 2. 多项目管理框架

### 2.1 项目生命周期

```
阶段0: 情报发现（Intelligence Discovery）
    Web Researcher持续监控 → 发现机会信号
    ↓
阶段1: 机会评估（Opportunity Assessment）
    Analyst + PM评估市场规模、竞争格局、技术可行性
    ↓
阶段2: 快速验证（Rapid Validation）
    Dev 48小时MVP → 市场测试 → 数据反馈
    ↓
阶段3: 决策点（Go/No-Go Decision）
    老板决策：投入 /  pivot /  放弃
    ↓
阶段4: 全面开发（Full Development）
    完整团队投入，3-4周发布
    ↓
阶段5: 运营增长（Operation & Growth）
    Ops + 增长团队持续运营优化
    ↓
阶段6: 退出/迭代（Exit/Iterate）
    出售 / 自动化运营 / 下线
```

### 2.2 项目组合管理

| 项目 | 阶段 | 状态 | 投入资源 | 预期回报 |
|------|------|------|----------|----------|
| Signal Hunter | 阶段5 | 🟢 运营中 | 20% | 验证团队 |
| Project X | 阶段1 | 🟡 评估中 | 10% | 待确定 |
| Project Y | 阶段0 | ⚪ 监控中 | 5% | 情报收集 |
| [储备] | - | ⚪ 等待 | 0% | - |

**资源分配原则**:
- 60% 当前主力项目（产生现金流）
- 30% 孵化新项目（增长曲线）
- 10% 情报探索（机会发现）

## 3. 情报战略系统

### 3.1 情报源矩阵

| 渠道 | 监控内容 | 频率 | 负责角色 | 工具 |
|------|----------|------|----------|------|
| **X/Twitter** | KOL动态、项目发布、情绪变化 | 实时 | Web Researcher | bird |
| **知乎** | 深度分析、趋势洞察、用户需求 | 每日 | Web Researcher | 手动+解析 |
| **微信** | 公众号、小程序动态、私域趋势 | 每日 | Web Researcher | RSS+监控 |
| **GitHub** | 开源项目、技术趋势、新工具 | 实时 | Web Researcher | API+RSS |
| **Discord/TG** | 社区动态、早期信号 | 实时 | Web Researcher | bot监控 |
| **ProductHunt** | 新产品发布、市场需求 | 每日 | Web Researcher | RSS |
| **行业报告** | 市场数据、投资趋势 | 每周 | Analyst | 爬虫+订阅 |

### 3.2 情报处理流程

```
多源情报输入
    ↓
[情报聚合层] Web Researcher 24/7监控
    ↓
[信号过滤层] Analyst识别高价值信号
    ↓
[机会评估层] PM + Analyst评估可行性
    ↓
[决策层] 老板决策：是否立项
    ↓
[执行层] 木木派遣团队快速验证
```

### 3.3 知识库自动更新

**触发条件**:
- 新技能发布 → 自动评估 → HR决定是否安装
- 新技术趋势 → Analyst研究 → 更新技术栈文档
- 竞品动态 → Web Researcher监控 → 更新竞品分析
- 用户反馈 → 汇总分析 → 更新产品roadmap

**自动化机制**:
```bash
# 每日情报收集 (cron定时任务)
0 9 * * * /skills/web-researcher/daily-intelligence.sh

# 生成情报摘要 → 发送给木木
# 木木审核 → 更新知识库 → 分发团队
```

## 4. 能力进化系统

### 4.1 技能持续更新机制

```
每周技能扫描（HR职责）
    ↓
skill_discovery.py --report
    ↓
识别新技能 → 评估价值 → 试用安装
    ↓
验证功能 → 整合到角色配置 → 团队培训
```

### 4.2 跨项目能力复用

**共享技能库**:
- 所有项目共享 `~/.openclaw/skills/` 全局技能
- 新项目自动继承成熟技能配置
- 项目专属技能安装在 `<project>/skills/`

**最佳实践沉淀**:
```
Project Alpha 成功经验
    ↓
提炼为通用模式 → 更新AI_TEAM_ARCHITECTURE.md
    ↓
Project Beta 继承应用 → 快速启动
```

### 4.3 团队学习机制

**每日情报简报** (Editor生成):
- X/Twitter热点摘要
- GitHub新项目发现
- 技术趋势洞察

**每周复盘** (全员参与):
- 本周学到了什么？
- 哪些技能提升了效率？
- 哪些流程需要优化？

**每月战略回顾** (老板+木木):
- 项目组合健康度
- 情报系统有效性
- 团队能力提升
