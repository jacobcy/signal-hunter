## 协作流程图

```
老板需求
    │
    ▼
┌────────────────┐
│  Step 1: 侦察   │ ← 派出PM/Analyst侦察小队
│  (5-10分钟)    │    产出：全景分析报告
└────────────────┘
    │
    ▼
┌────────────────┐
│  Step 2: 规划   │ ← 木木按依赖拆任务
│  (5分钟)       │    产出：任务队列+分配策略
└────────────────┘
    │
    ▼
┌────────────────────────────────────────────────────────────┐
│  Step 3: 实施（并行30-60分钟）                              │
│  同时派遣：                                                  │
│    ├─ Dev小队: 代码实现                                      │
│    ├─ QA小队:  同步编写测试                                  │
│    ├─ Editor:  同步编写文档                                  │
│    └─ Ops:     准备部署脚本                                  │
└────────────────────────────────────────────────────────────┘
    │
    ▼
┌────────────────┐
│  Step 4: 整合   │ ← 收集结果、解冲突、合并、验证
│  (10-20分钟)   │
└────────────────┘
    │
    ▼
┌────────────────────────────────────────────────────────────┐
│  Step 5: 汇报（给老板）                                      │
│  高密度一句话：做了什么 | 发现什么 | 还差什么                  │
└────────────────────────────────────────────────────────────┘
```
